But first of all, let us quickly explain how Datastringer works.

Getting started

There are three categories of things in Datastringer:

  • Stringers: a stringer is essentially a function. It takes some parameters. With these, it will get and crunch some data and tell wether there's an alert.
  • Use-cases: a use-case combines a stringer's name and set of parameters to feed it.
  • Datastringer itself, reads a list of use-cases from a file. For each use-case, it runs the stringer with the given parameters, and send an email if there's an alert.

What we're going to do

Datastringer comes with two built-in stringers. Both are based on the fantastic Police.uk open-government API. We are going two build to example use-cases, one for each.

  • The first example will monitor a local police force, by occasionally checking if a new bobby has been attached to the force, or for planned events for the community, or for changes in your local force priorities
  • The second one will alert you in case of sudden rise or fall in crime in a neighbourhood. This will be done by fetching a bunch of statistics from Police.uk, mashing them together, calculating averages and deltas... and finally checking if the thresholds you defined were crossed.

You can (and should) read more about the examples on the Wiki.
